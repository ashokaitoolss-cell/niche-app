# Deployment

The backend needs to run 24/7 for the hourly job, so it needs a host that keeps a process alive
(not a serverless function host). The frontend is a static PWA and can go anywhere that serves static files.

## Backend — Railway (recommended)

Railway's free/hobby tier keeps a small always-on service running, which is what the hourly
APScheduler job needs.

1. `railway init` from `backend/`, or connect the repo in the Railway dashboard and set the root
   directory to `backend/`.
2. Railway auto-detects Python via Nixpacks and reads `railway.toml` for the start command and
   health check (`/api/health`).
3. Add a Railway **volume** mounted at `/data`, and set the env var `DATABASE_PATH=/data/niche.db`
   so the SQLite file survives redeploys.
4. Set the remaining env vars from `.env.example` in the Railway service settings
   (`ANTHROPIC_API_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, and the optional Product Hunt /
   Crunchbase keys).
5. Set `CORS_ORIGINS` to the deployed frontend URL once you have it.

Render (web service, not static site) or Fly.io work the same way — persistent disk +
`uvicorn app.main:app --host 0.0.0.0 --port $PORT` as the start command. Fly.io needs a `fly.toml`
(`fly launch` will generate one from the Dockerfile-less Python app via its Python builder).

## Frontend — Vercel (recommended)

1. Import the repo in Vercel, set the root directory to `frontend/`.
2. `vercel.json` already sets the build command (`npm run build`) and output dir (`dist`).
3. Set the env var `VITE_API_BASE_URL` to the deployed backend URL (e.g.
   `https://niche-backend.up.railway.app`).
4. Vercel serves the PWA manifest and service worker (generated by `vite-plugin-pwa`) as static
   assets automatically — no extra config needed for installability.

Netlify works identically (point the build command / publish directory at the same values).

## After first deploy

- Message your Telegram bot `/idea` to confirm the daily synthesis job can reach both the DB and
  the Anthropic API.
- Watch the first hourly run's logs to confirm at least one of the RSS/API sources returns data —
  some sources (Product Hunt, Crunchbase) silently no-op without their optional API keys.
